Objetivo: Demonstrar diferentes formas de carregar e customizar recursos (CSS, JavaScript, imagens) no Apache Wicket de maneira simples e prática.
1. ResourceReference (Arquivos Externos)

Clique Aqui!

Este box usa CSS e JS carregados via ResourceReference

Como funciona: 
private static final CssResourceReference CUSTOM_CSS = 
    new CssResourceReference(MinhaClasse.class, "custom-resource.css");

response.render(CssHeaderItem.forReference(CUSTOM_CSS));
2. CSS Inline (Direto no Código)
CSS adicionado diretamente no Java!
Como funciona: 
String css = ".my-class { color: red; }";
response.render(CssHeaderItem.forCSS(css, "my-css-id"));
3. JavaScript Inline
Como funciona: 
String js = "function minhaFuncao() { alert('Olá!'); }";
response.render(JavaScriptHeaderItem.forScript(js, "my-js-id"));
4. Recursos de CDN
Animação de CDN!
Este box usa Animate.css carregado de CDN
Como funciona: 
response.render(CssHeaderItem.forUrl(
    "https://cdn.../animate.min.css", 
    "animate-css"
));
5. Context Relative Resources (webapp/)

Logo carregado de src/main/webapp/logo.png

Como funciona: 
// Para recursos em src/main/webapp/
new ContextRelativeResourceReference("css/style.css");

// Ou simplesmente use tag HTML normal
<img src="logo.png" />
Dicas Práticas
Boas Práticas
  • Declare ResourceReferences como static final
  • Use versionamento em produção
  • Coloque recursos ao lado das classes
  • Use minificação em produção
  • CDN para bibliotecas populares
Evite
  • Criar ResourceReferences em loops
  • Carregar recursos desnecessários
  • Cache agressivo em desenvolvimento
  • Misturar muitos estilos inline
  • Esquecer de testar em produção
